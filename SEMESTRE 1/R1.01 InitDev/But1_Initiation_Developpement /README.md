# Initiation au Développement — R1.01

Ce dossier regroupe les ressources pour la matière d'initiation au développement (algorithmique et premières notions de programmation). Il est structuré pour accompagner la progression : cours, exercices guidés, TP, SAE et corrigés.

## Objectifs détaillés
- Maîtriser les concepts d'algorithmique élémentaire : variables, expressions, affectation.
- Utiliser les structures de contrôle : conditionnelles (`if/else`), boucles (`for`, `while`).
- Concevoir et factoriser du code via fonctions/procédures.
- Manipuler des tableaux et comprendre la complexité élémentaire des algorithmes (notations O).
- Traduire un énoncé algorithmique en code (pseudo-code -> langage cible).

## Plan de cours (suggestion)
1. Notions de base et variables
2. Expressions et opérateurs
3. Structures conditionnelles
4. Boucles et itérations
5. Tableaux et parcours
6. Fonctions / procédures
7. Tri et recherche simples (tri par sélection, insertion, recherche linéaire/bin.)
8. Notions de complexité (O(n), O(n^2))
9. Projet SAE : analyser un énoncé, concevoir une solution et implémenter

## Contenu du dossier
- `Partie 1 - Algorithmique/` — supports de cours et exercices corrigés.
- `PARTIE2/` — compléments et implémentations en langage étudié.
- `SAE/` — énoncés de projets (sujets, livrables attendus, critères d'évaluation).
- `corrigé/` — solutions et corrections détaillées.

## Ressources et outils recommandés
- Environnement : éditeur VS Code, compilateur/interpréteur selon le langage (ex : Python). 
- Outils : Git pour versionner les TP, un tableur pour organiser jeux de données simples.
- Référence rapide : pseudocode + exemples fournis dans les PDFs du dossier.

## Exercices pratiques (exemples)
- Écrire une fonction qui calcule la factorielle.
- Implémenter un tri par insertion et mesurer le temps sur listes de tailles croissantes.
- Résoudre l'énoncé SAE : décomposer en modules, écrire des tests simples.

## Conseils d'apprentissage
- Faire d'abord la version papier (pseudocode), puis coder.
- Tester avec cas simples puis cas limites (liste vide, taille 1, valeurs répétées).
- Documenter les choix architecturaux pour la SAE.

## Évaluations
- TD/TP notés : suivre les consignes et fournir dépôt Git ou archive selon l'énoncé SAE.

_Dernière mise à jour :_ Septembre 2025
